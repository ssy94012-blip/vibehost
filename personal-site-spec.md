# 퍼스널 홍보 사이트 스펙

## 1. 프로젝트 개요

- 목적: 나를 소개하고, 이력/포트폴리오/취미를 한 번에 보여주는 개인 홍보 사이트
- 페이지 형태: 한 페이지로 길게 스크롤되는 싱글 페이지
- 핵심 분위기: 하늘색 기반, 부드럽고 깨끗한 이미지, 너무 딱딱하지 않은 감성
- 필수 메뉴: 홈, 자기소개, 포트폴리오, 취미
- 추가 기능: 방문자 수 카운트 표시

## 2. 디자인 방향

- 메인 컬러: 하늘색, 밝은 흰색, 아주 연한 회색
- 포인트 컬러: 민트 또는 네이비 소량 사용
- 전체 무드: 부드러운 그라데이션, 둥근 카드, 은은한 그림자, 여백이 넉넉한 구성
- 이미지 스타일: 밝고 따뜻한 느낌, 과하지 않은 블러/그라데이션 배경 활용
- 스크롤 경험: 메뉴 클릭 시 부드럽게 이동, 섹션 등장 애니메이션은 은은하게 적용

## 3. 화면 구조

### 홈

- 첫 화면에서 이름, 한 줄 소개, 대표 이미지 느낌이 보이도록 구성
- 방문자 수 카운트를 눈에 띄지만 과하지 않게 표시
- 아래로 스크롤을 유도하는 버튼 또는 아이콘 배치

### 자기소개

- 내 사진 1장
- 짧은 자기소개 문장
- 이력(학력, 경력, 활동, 자격/수상 등) 정리
- 강점 또는 키워드 3~5개 노출

### 포트폴리오

- 총 4개 프로젝트 카드 또는 섹션
- 각 항목에 썸네일, 제목, 설명, 역할, 사용 기술, 링크 포함

### 취미

- 총 3개 취미 소개
- 각 취미마다 사진 또는 아이콘, 짧은 설명, 왜 좋아하는지 한 줄 포함

## 4. 상세 요구사항

### 공통

- PC와 모바일 모두 자연스럽게 보여야 함
- 상단 메뉴는 각 섹션으로 이동하는 앵커 방식
- 전체 페이지는 세로 스크롤 중심
- 섹션 간 구분은 배경톤 변화나 곡선형 구획으로 부드럽게 처리

### 방문자 수 카운트

- 홈 섹션에 `총 방문 수` 표시
- 가능하면 푸터에도 한 번 더 작게 표시
- 실제 방문 수 집계를 위해 서버리스 함수 또는 외부 DB 연동 필요
- 임시 데모는 `localStorage`로 가능하지만, 실제 전체 방문자 수 집계는 불가능

## 6. 문구 스타일 가이드

- 너무 딱딱한 이력서 말투보다, 친근하지만 정돈된 말투 사용
- 소개 문장은 짧고 선명하게 작성
- 포트폴리오 설명은 "무엇을 만들었는지 + 어떤 역할을 했는지 + 어떤 결과가 있었는지" 순서 권장
- 취미 소개는 사람적인 매력이 느껴지도록 부드럽게 작성

## 7. 구현 메모

- 싱글 페이지 섹션 순서 권장:
  `홈 -> 자기소개 -> 포트폴리오 -> 취미 -> 푸터`
- 방문자 수는 추후 구현 시 아래 방식 중 하나 선택
  - 권장: Supabase + 서버리스 함수
  - 대안: Firebase
  - 간단한 데모: `localStorage`
- 포트폴리오와 취미 이미지는 직사각형 카드보다, 둥근 모서리의 큰 이미지 카드가 잘 어울림

## 8. 완료 기준

- 홈, 자기소개, 포트폴리오, 취미 메뉴가 모두 보일 것

- 하늘색 기반의 부드러운 디자인일 것
- 자기소개에 사진과 이력이 포함될 것
- 포트폴리오 4개가 모두 들어갈 것
- 취미 3개가 모두 들어갈 것
- 방문자 수 카운트가 화면에 표시될 것
